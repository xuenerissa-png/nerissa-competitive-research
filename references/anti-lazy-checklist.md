# Anti-Lazy Pre-Publish Checklist

> Before any competitor research is published / sent to team / used in pitch deck — run this checklist. Don't ship until all green.

## Funding Data (most common failure point)

- [ ] Cumulative funding figure cites ≥ 2 independent sources
- [ ] Sources include at least one of: Crunchbase / TechCrunch / official PR / SEC filing / PitchBook / LATKA
- [ ] If the company is bootstrapped, LATKA or founder direct statement is cited (don't trust aggregator blogs alone)
- [ ] Latest round date is within last 2 years OR clearly marked as "old data"
- [ ] Valuation, if cited, comes from official PR not third-party speculation

## Website Verification

- [ ] I (or Claude) opened the official website in the last 24 hours
- [ ] If site returned 403 / 404 / didn't load → that fact is documented, NOT silently assumed as "no website"
- [ ] At least 3 pages were viewed: home / pricing (or contact-sales) / customers (or case studies)
- [ ] Screenshots saved (optional but recommended for audit trail)

## Essence Decomposition

- [ ] I asked "what is the essence" — not just "what does it do"
- [ ] One of: technology / packaging-layer / sales motion / brand / network effect is identified
- [ ] If it's a packaging layer (composed of other tools), the underlying tools are named
- [ ] The moat type is identified (technology / scale / network / brand / capital)

## Customer Overlap

- [ ] Customer overlap was VERIFIED — not estimated by gut
- [ ] Sources checked: official customer wall, ≥ 1 case study, Twitter / G2 / ProductHunt reviews
- [ ] If competitor's main customer profile doesn't match yours, this is the FIRST line of the recommendation (not buried)
- [ ] If they have giant logos that don't reflect your reality, that's noted (e.g. "they show F500 but their actual ICP is mid-market")

## Inheritance Discipline

- [ ] If this research builds on a previous DD / report, I explicitly re-validated each claim
- [ ] If I didn't re-validate something, it's marked with `[NOT RE-VALIDATED]`
- [ ] I did NOT silently inherit funding figures, customer counts, or judgments from past reports

## Dual Track Discipline

- [ ] I labeled which track this analysis is for: A · Fundraising Reference / B · Business Learning Target
- [ ] If Track A, the company has ≥ $10M funding in similar space
- [ ] If Track B, the team / capital / path is realistically replicable
- [ ] I did not mix tracks (e.g. killing a 0-funding company because "our goal is $10M" — that's Track A logic killing a Track B candidate)

## Anti-Hallucination

- [ ] Every "they signed X customers" / "they have Y users" claim cites a source
- [ ] If your own company's "we signed X" or "we have Y" appears, that is verified against source-of-truth database (not memory)
- [ ] If a 1-sample observation is being generalized to a vertical, the generalization is explicitly questioned

## Anti-Free-Advertising

- [ ] No "anti-[competitor name]" framing (free advertising for them)
- [ ] No reverence-tone reproducing competitor marketing copy without critique
- [ ] If a giant (Alibaba / ByteDance / Meta / Google) is mentioned, threat is framed neutrally (not "they're amazing", not "they're stupid")

## Output Discipline

- [ ] All 15 mandatory fields filled
- [ ] One-line "what it is" and one-line "what to copy" are sharp, not vague
- [ ] 3 must-see pages are pickable URLs (not "go look around")
- [ ] Data sources field has ≥ 2 URLs (clickable)
- [ ] If any cells say "TBD" or "unknown" — that's marked clearly, not hidden

## Sign-off

If all boxes ticked: 🟢 Ready to publish.
If any unticked: 🔴 Go fix first. Don't ship sloppy research.
